## **Problem: September Account Performance Analytics**

A regional technician service company maintains two separate datasets to manage its operations. They have a history of service transactions dating back to 2022 and need to generate a specific financial performance report for the month of **September 2022**.

Your task is to write a query that joins these datasets to calculate key transaction metrics for every account that was active during that period.

---

### **Input Format**

#### **TABLE: `account`**
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | Integer | The unique identifier for the account. |
| `ibmn` | String | The International Business Management Number assigned to the account. |

#### **TABLE: `transaction`**
| Column | Type | Description |
| :--- | :--- | :--- |
| `id` | Integer | The identifier linking back to the `account` table. |
| `datetime` | Timestamp | The date and time the transaction occurred. |
| `amount` | Decimal | The monetary value of the transaction. |

---

### **Task**
Generate a report that contains the following columns for transactions occurring in **September 2022**:
1.  **`ibmn`**: The account's management number.
2.  **`min_amount`**: The minimum transaction amount for that account in September.
3.  **`max_amount`**: The maximum transaction amount for that account in September.
4.  **`average_amount`**: The average transaction amount for that account in September (rounded to 2 decimal places).
5.  **`total_amount`**: The sum of all transaction amounts for that account in September.

---

### **Constraints**
* Only transactions where the `datetime` is between **2022-09-01** and **2022-09-30** (inclusive) should be included.
* The output must include the `id` from the account table to ensure data integrity.
* The final results must be sorted in **ascending order** by the `ibmn`.
* Ensure all numerical values are reported, even if they are zero.

---

### **Output Format**
The output should be a table with five columns: `ibmn`, `min_amount`, `max_amount`, `average_amount`, and `total_amount`.

| ibmn | min_amount | max_amount | average_amount | total_amount |
| :--- | :--- | :--- | :--- | :--- |
| IBMN001 | 50.00 | 500.00 | 275.00 | 550.00 |
| IBMN002 | 10.00 | 100.00 | 55.00 | 110.00 |
